# Patches

Applied to `kubernetes-csi/external-resizer` v2.1.0 (k8s 1.34 track)
during build of the `csi-resizer` binary.

## 001-fix-cve.patch

Bumps direct/indirect dependencies in `go.mod` / `go.sum` to versions
that fix CVEs reported by Trivy on the v2.1.0 binary:

- `google.golang.org/grpc` `v1.78.0 -> v1.80.0` (fixes CVE-2026-33186)
- `go.opentelemetry.io/otel` `v1.39.0 -> v1.43.0` (fixes CVE-2026-29181)
- `go.opentelemetry.io/otel/sdk` `v1.39.0 -> v1.43.0` (fixes
  CVE-2026-24051, CVE-2026-39883)
- `golang.org/x/crypto` `v0.46.0 -> v0.47.0`

Generated by cloning v2.1.0, running `go mod edit -require=...` and
`go mod tidy -e`, then `git format-patch -1`. Only touches `go.mod` /
`go.sum`; the build pipeline does `rm -r vendor && go mod download`
before `make build`. Verified end-to-end inside `golang:1.25-alpine`
with `go build -mod=readonly`.

## 002-fix-cve-golang-x.patch

Bumps `golang.org/x/net` -> `v0.56.0` (CVE-2026-46600) and
`golang.org/x/text` -> `v0.39.0` (CVE-2026-56852) in `go.mod` / `go.sum` to
clear the findings reported by Trivy; this also pulls the transitive
`golang.org/x/crypto` -> `v0.53.0`, `golang.org/x/sys` -> `v0.46.0`,
`golang.org/x/sync` -> `v0.21.0` and `golang.org/x/term` -> `v0.44.0` bumps. Generated by cloning
the pinned upstream version, applying the existing patches, then
`go get`/`go mod tidy` and `git format-patch -1`; touches only
`go.mod` / `go.sum` (the build re-runs `go mod vendor`/`download`).
